## 🧠 Topics

| Module | What's Inside |
|--------|--------------|
| **Recursion** | Factorial, Fibonacci (two flavours), GCD, sum of digits, prime sums, Tower of Hanoi |
| **Brute Force** | Convex hull (gift wrapping), minimum pairwise difference, word search |
| **Searching** | Binary search — recursive and iterative, index-based variants |
| **OOP** | Classes, encapsulation, operator overloading in realistic C++ scenarios |
| **Linked Lists** | Node creation, insertion, reversal, min/max traversal, sentinel-based input |
| **Stacks** | Bracket evaluation, operator precedence eval, infix → postfix, both array and linked-list backed implementations |
| **Queues** | Full queue ADT from scratch |
| **Interview Prep** | Handpicked problems from actual coding rounds |

## ⚡ Why C++ and not Python or Java?

> *"The only way to learn a new programming language is by writing programs in it. In C++, you also learn how computers actually work."*
>
> — Bjarne Stroustrup

Raw speed. Fine-grained memory control. Pointer arithmetic when you need it. Deterministic behavior. No garbage collector pausing mid-contest. Java wraps everything in objects you didn't ask for. Python hands you a spoon when you need a scalpel.

**C++ trusts you. And if you're serious about DSA — that trust is the whole point.**

## 🚀 Getting Started

```bash
# 1. Clone
git clone https://github.com/antoniusjairus4/DSA.git
cd DSA

# 2. Compile anything
g++ -std=c++17 -O2 -o out DSA/linked_lists/insertion.cpp

# 3. Run
./out

# No npm install. No pip install. No venv. No Makefile.
# Just a compiler and curiosity.
```
